\section{Conclusiones}

Después de haber realizado la práctica se puede concluir que:

Para el caso del controlador de versiones:
\begin{itemize}
	\item Los controladores de versiones son muy importantes en cualquier proyecto de desarrollo de software pues facilita tener a disposición cualquier versión y controlar los cambios que se han realizado en el proyecto.	
	\item Git es un controlador de versiones fácil de instalar y manejar, aunque se realiza por consola los pasos prácticamente son muy intuitivos y se obtienen resultados muy claros.
	\item La integración con github ofrece facilidad de publicar proyectos de desarrollo disponibles para toda la comunidad.
	\item El hecho que para poder configurar los usuarios permitidos en los proyectos Github requiera pagos implica un inconveniente en el caso que se desee restrigir o controlar los cambios que se realizan en el proyecto y las copias y ventas que se hagan con el, es recomendable marcar cada componente del proyecto con la licencia que ayude a protegerlo.
	\item Git y github tienen ciertos inconvenientes con las modificaciones al tiempo que realicen los colaboradores pero ofrece soluciones de permitir guardar los cambios detectados aunque el desarrollador deba modificarlo manualmente.
\end{itemize}

Para el caso de la documentación:
\begin{itemize}
\item Es fácil obtener un manual/documentación estructurado en html de un desarrollo y siendo así es facil incluirla en cualquier desarrollo.
\item Al igual de latex se debe tener conocimiento de las etiquetas y del lenguaje manejado, aunque se encuentra muy bien documentado en su página principal.
\item La parte gráfica facilita la compilación aunque también se puede realizar por consola.
\end{itemize}

Para el caso de latex:
\begin{itemize}
\item Es estable, multiplataforma, es gratuito y permite estructurar fácilmente los documentos 
\item Se debe tener especial cuidado en la forma de poner las etiquetas, pues una etiqueta mal cerrada, un parámetro mal escrito implica un error en la compilación y no será posible ver el documento.
\item La curva de aprendizaje es bastante amplia, al no ser tan gráfico como los demás procesadores presenta un grado de dificultad.
\item Hasta que no se compile no se tiene una visón del documento, lo que representa una dificultad y requiere de tiempo en caso que no se tenga habilidad se demorá más en generar un documento.
\item Los gráficos no se insertan tan fácilmente, presenté dificultad respecto a ubicarlos donde quería en este caso debería tener en el begin{figure} el parámetro H entre corchetes.

\end{itemize}
